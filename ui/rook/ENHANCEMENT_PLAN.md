# Rook Tauri Enhancement Plan

This file is no longer the source of truth for feature status.

Use [../ROOK_FEATURE_BLEND_LEDGER.md](../ROOK_FEATURE_BLEND_LEDGER.md) instead.

## Why

The old plan drifted behind the feature branch and was still describing already-landed work such as:
- tray
- deep links
- global shortcuts

The current phase is feature blending and stabilization, not broad migration planning.

## Current Rule

- Track what is actually implemented in the ledger.
- Mark manual verification honestly.
- Only add new features after the shell behavior and golden provider/chat flow are stable.
- [ ] Notifications - appear and are clickable
- [ ] Zoom - Ctrl/Cmd +/- works
- [ ] Keyboard shortcuts - all work
- [ ] Providers - each provider connects
- [ ] Chat - send/receive messages
- [ ] Animations - smooth and performant
- [ ] Window state - position/size restored
