In our previous episode of [rook Flight School](https://www.youtube.com/playlist?list=PLyMFt_U2IX4s1pMaidir5P4lSfjUK6Nzm), our host [Adewale Abati](https://www.linkedin.com/in/acekyd/) showed us how to use rook to take a Figma design and transform it into a functional Nuxt application. In this stream he covered the entire process, from initial set up to final implementation, highlighting how rook can help developers bridge the gap between design and development.

<!--truncate-->

# How It Works
[Extensions](https://rook-docs.ai/docs/getting-started/using-extensions) enhance rook's functionality by connecting with your existing tools and workflows. They add new features, access external data resources, and integrate with other systems. Learn how multiple extensions, including Figma and Developer, worked together seamlessly to dramatically accelerate development.

# Live Tutorial: rook Builds Live
During the livestream, Adewale demonstrated step-by-step how rook handled each development phase, from creating the basic application structure to generating responsive layouts using Tailwind CSS. Adewale also highlighted how rook addresses potential limitations as you go, showcasing the powerful balance between the rook's automation and developer control.

Throughout the stream, Adewale shared valuable tips to prepare your design for rook. His key recommendations include: 

* start with a well-structured Figma design
* use rook to make targeted improvements after the initial generation
* fine-tune specific elements as needed
* make sure you thoroughly test for functionality and responsiveness

# Getting Started with rook and Figma
Whether you're a designer wanting to rapidly turn concepts into working code or a developer curious about streamlining design implementation, you can download rook with its built-in [Developer extension](https://rook-docs.ai/docs/getting-started/installation) and add the [Figma extension](https://rook-docs.ai/v1/extensions/). 

For step-by-step instructions, check out the [Figma tutorial](/docs/mcp/figma-mcp).
